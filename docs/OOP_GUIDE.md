# 心语编程语言 - 面向对象编程指南

## 概述

心语编程语言v3.1引入了完整的面向对象编程（OOP）支持，包括类、继承、接口、多态等特性。本指南将详细介绍如何使用这些特性。

## 类定义

### 基本类定义

```心语
定义 类 动物：
    定义 名字 = ""。
    定义 年龄 = 0。

    函数 初始化(参数 名字, 参数 年龄)：
        自身.名字 = 名字。
        自身.年龄 = 年龄。
    。

    函数 叫()：
        返回 "动物叫声"。
    。

    函数 介绍()：
        打印 "我是" 自身.名字 "，今年" 自身.年龄 "岁。"。
    。
。
```

### 创建对象

```心语
定义 我的宠物 = 新建 动物("小白", 3)。
我的宠物.介绍()。  # 输出：我是小白，今年3岁。
打印 我的宠物.叫()。  # 输出：动物叫声
```

## 继承

### 单继承

```心语
定义 类 狗 继承 动物：
    函数 叫()：
        返回 "汪汪"。
    。

    函数 摇尾巴()：
        打印 自身.名字 "正在摇尾巴..."。
    。
。

定义 我的狗 = 新建 狗("旺财", 2)。
我的狗.介绍()。  # 输出：我是旺财，今年2岁。
打印 我的狗.叫()。  # 输出：汪汪
我的狗.摇尾巴()。  # 输出：旺财正在摇尾巴...
```

### 方法重写

子类可以重写父类的方法：

```心语
定义 类 猫 继承 动物：
    函数 叫()：
        返回 "喵喵"。
    。

    函数 抓老鼠()：
        打印 自身.名字 "正在抓老鼠..."。
    。
。
```

## 接口和多态

### 接口定义

```心语
定义 接口 可发声：
    函数 发声()。
。

定义 接口 可移动：
    函数 移动()。
。
```

### 实现接口

```心语
定义 类 鸟 实现 可发声, 可移动：
    函数 发声()：
        返回 "叽叽喳喳"。
    。

    函数 移动()：
        返回 "飞翔"。
    。
。
```

### 多态使用

```心语
定义 动物列表 = 列表(新建 狗("小黑", 1), 新建 猫("小花", 2), 新建 鸟())。
循环 动物 遍历 动物列表：
    打印 动物.叫()。
。
```

## 静态成员

### 静态属性和方法

```心语
定义 类 数学工具：
    静态 定义 圆周率 = 3.14159。

    静态 函数 平方(参数 x)：
        返回 x 相乘 x。
    。

    静态 函数 最大值(参数 a, 参数 b)：
        如果 a 大于 b 那么：
            返回 a。
        否则：
            返回 b。
        。
    。
。

打印 数学工具.圆周率。  # 输出：3.14159
打印 数学工具.平方(5)。  # 输出：25
打印 数学工具.最大值(10, 20)。  # 输出：20
```

### 类方法

```心语
定义 类 工厂：
    类函数 创建实例(参数 值)：
        返回 工厂(值)。
    。

    函数 初始化(参数 值)：
        自身.值 = 值。
    。

    函数 显示()：
        打印 "工厂实例，值：" 自身.值。
    。
。

定义 实例 = 工厂.创建实例(100)。
实例.显示()。  # 输出：工厂实例，值：100
```

## 访问控制

### 公共成员（默认）

所有成员默认都是公共的：

```心语
定义 类 示例：
    定义 公共属性 = "公共"。

    函数 公共方法()：
        返回 "公共方法"。
    。
。
```

### 私有成员约定

使用下划线前缀表示私有成员（约定）：

```心语
定义 类 银行账户：
    定义 _余额 = 0。

    函数 初始化(参数 初始余额)：
        自身._余额 = 初始余额。
    。

    函数 存款(参数 金额)：
        自身._余额 = 自身._余额 相加 金额。
    。

    函数 取款(参数 金额)：
        如果 金额 小于等于 自身._余额 那么：
            自身._余额 = 自身._余额 相减 金额。
            返回 真值。
        否则：
            返回 假值。
        。
    。

    函数 查询余额()：
        返回 自身._余额。
    。
。
```

## 完整示例

### 电子商务系统示例

```心语
# 定义接口
定义 接口 可支付：
    函数 支付(参数 金额)。
。

定义 接口 可发货：
    函数 发货()。
    。
    函数 跟踪()。
。

# 定义基类
定义 类 商品：
    定义 名称 = ""。
    定义 价格 = 0。
    定义 库存 = 0。

    函数 初始化(参数 名称, 参数 价格, 参数 库存)：
        自身.名称 = 名称。
        自身.价格 = 价格。
        自身.库存 = 库存。
    。

    函数 显示信息()：
        打印 "商品：" 自身.名称。
        打印 "价格：" 自身.价格 "元"。
        打印 "库存：" 自身.库存 "件"。
    。

    函数 减少库存(参数 数量)：
        如果 数量 小于等于 自身.库存 那么：
            自身.库存 = 自身.库存 相减 数量。
            返回 真值。
        否则：
            打印 "库存不足！"。
            返回 假值。
        。
    。
。

# 定义具体商品类
定义 类 图书 继承 商品 实现 可支付, 可发货：
    定义 作者 = ""。
    定义 ISBN = ""。

    函数 初始化(参数 名称, 参数 价格, 参数 库存, 参数 作者, 参数 ISBN)：
        父类.初始化(名称, 价格, 库存)。
        自身.作者 = 作者。
        自身.ISBN = ISBN。
    。

    函数 显示信息()：
        父类.显示信息()。
        打印 "作者：" 自身.作者。
        打印 "ISBN：" 自身.ISBN。
    。

    函数 支付(参数 金额)：
        如果 金额 大于等于 自身.价格 那么：
            打印 "支付成功！"。
            返回 真值。
        否则：
            打印 "支付金额不足！"。
            返回 假值。
        。
    。

    函数 发货()：
        打印 "图书" 自身.名称 "已发货。"。
    。

    函数 跟踪()：
        打印 "图书" 自身.名称 "运输中..."。
    。
。

# 定义订单类
定义 类 订单：
    静态 定义 订单计数 = 0。

    定义 订单号 = ""。
    定义 商品列表 = 列表()。
    定义 总金额 = 0。

    函数 初始化()：
        自身.订单号 = "ORD" 相加 转字符串(订单.订单计数)。
        订单.订单计数 = 订单.订单计数 相加 1。
    。

    函数 添加商品(参数 商品, 参数 数量)：
        如果 商品.减少库存(数量) 那么：
            自身.商品列表.追加((商品, 数量))。
            自身.总金额 = 自身.总金额 相加 (商品.价格 相乘 数量)。
            打印 "已添加" 数量 "件" 商品.名称。
        。
    。

    函数 结算()：
        打印 "订单号：" 自身.订单号。
        打印 "商品列表："。

        循环 (商品, 数量) 遍历 自身.商品列表：
            打印 "  - " 商品.名称 " x " 数量 " = " (商品.价格 相乘 数量) "元"。
        。

        打印 "总金额：" 自身.总金额 "元"。
    。
。

# 使用示例
定义 图书1 = 新建 图书("Python编程", 59.9, 100, "张三", "978-7-123-45678-9")。
定义 图书2 = 新建 图书("心语语言指南", 39.9, 50, "李四", "978-7-987-65432-1")。

图书1.显示信息()。

定义 我的订单 = 新建 订单()。
我的订单.添加商品(图书1, 2)。
我的订单.添加商品(图书2, 1)。
我的订单.结算()。

如果 图书1.支付(119.8) 那么：
    图书1.发货()。
    图书1.跟踪()。
。
```

## 最佳实践

### 1. 类设计原则

- **单一职责原则**：每个类应该只有一个改变的理由
- **开放封闭原则**：对扩展开放，对修改封闭
- **里氏替换原则**：子类应该可以替换父类
- **接口隔离原则**：使用多个专门的接口而不是一个通用接口
- **依赖倒置原则**：依赖抽象而不是具体实现

### 2. 命名约定

- 类名使用大驼峰命名法：`动物`、`银行账户`、`订单管理`
- 方法名使用小驼峰命名法：`初始化`、`显示信息`、`计算总额`
- 属性名使用小驼峰命名法：`商品名称`、`订单金额`、`用户列表`
- 私有成员使用下划线前缀：`_内部数据`、`_缓存`

### 3. 继承 vs 组合

- 使用继承表示"是一个"关系
- 使用组合表示"有一个"关系
- 优先使用组合而不是继承

### 4. 接口设计

- 接口应该小而专注
- 避免"上帝接口"
- 使用接口实现多态

## 常见问题

### 1. 如何访问父类方法？

使用`父类`关键字：

```心语
定义 类 子类 继承 父类：
    函数 重写方法()：
        # 调用父类方法
        父类.重写方法()。
        # 添加子类逻辑
        打印 "子类逻辑"。
    。
。
```

### 2. 如何实现抽象类？

使用接口和空实现：

```心语
定义 接口 抽象类：
    函数 抽象方法()。
。

定义 类 具体类 实现 抽象类：
    函数 抽象方法()：
        # 具体实现
        打印 "具体实现"。
    。
。
```

### 3. 如何处理多重继承？

心语支持多重接口实现，但不支持多重类继承：

```心语
定义 接口 A：
    函数 方法A()。
。

定义 接口 B：
    函数 方法B()。
。

定义 类 C 实现 A, B：
    函数 方法A()：
        打印 "实现A"。
    。

    函数 方法B()：
        打印 "实现B"。
    。
。
```

## 总结

心语编程语言的OOP系统提供了完整的面向对象编程支持，包括：

1. **类定义和对象创建**
2. **继承和方法重写**
3. **接口和多态**
4. **静态成员和类方法**
5. **模块导入导出**
6. **访问控制约定**

这些特性使得心语语言能够构建复杂的大型应用程序，同时保持代码的可维护性和可扩展性。
