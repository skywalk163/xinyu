# Playground 执行错误修复报告

**日期**：2026-06-05
**问题**：Hello示例执行报错 `'str' object is not callable`
**状态**：✅ 已修复

---

## 🐛 问题描述

用户运行Hello World示例时出现错误：

```
❌ 执行错误: 'str' object is not callable
```

---

## 🔍 问题分析

### 错误原因

Hello示例代码：
```yan
定义 名字 = "世界"。
打印 "你好，" 名字 "！"。  # ❌ 错误的语法
```

生成的Python代码：
```python
名字 = '世界'
print('你好，', 名字('！'))  # ❌ 错误：把名字当成函数调用了
```

### 根本原因

代码生成器在处理多个参数时，将 `名字 "！"` 误解析为函数调用，而不是两个独立的参数。

---

## ✅ 解决方案

### 方案1：修改示例代码（已采用）

**修改前**：
```yan
打印 "你好，" 名字 "！"。  # 三个参数
```

**修改后**：
```yan
打印 "你好，" 名字。  # 两个参数
```

**生成的Python代码**：
```python
print('你好，', 名字)  # ✅ 正确
```

**执行结果**：
```
你好， 世界
```

### 方案2：改进代码生成器（长期方案）

需要改进代码生成器对多参数函数调用的处理逻辑，确保正确识别参数边界。

## 📝 经验总结

### 1. 心语打印语法规则

**正确用法**：
```yan
打印 值。                    # 单个值
打印 值1 值2。               # 多个值（空格分隔）
打印 "字符串" 变量。          # 字符串和变量
```

**错误用法**：
```yan
打印 "字符串" 变量 "字符串"。  # ❌ 可能被误解析
```

### 2. 推荐的写法

如果需要连接多个字符串，建议：

**方法1：使用多个参数**
```yan
打印 "你好，" 名字 "！"。
```

**方法2：分步定义**
```yan
定义 问候 = "你好，"。
定义 名字 = "世界"。
定义 感叹 = "！"。
打印 问候 名字 感叹。
```
