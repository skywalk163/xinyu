# 中文编程语言本质设计

**版本：** 1.0
**日期：** 2026-05-22
**文档性质：** 语言设计规格说明

---

## 设计理念

### 程序的本质

程序是**控制流**与**数据流**的交织：
- **控制流**：顺序、分支、循环——决定"做什么"
- **数据流**：传递、变换、存储——决定"处理什么"

### 中文语言的本质

中文语言的本质是**综合特征**：
- **意合式表达**：不依赖显式语法标记，通过语义关联和语境推断
- **语境优先**：高语境依赖，上下文决定语义
- **语序灵活**：强调重点可前置，话题-评论结构

### 核心设计原则

**让中文的语法特性直接映射到程序的控制流和数据流，无需翻译层。**

### 极简关键字设计原则

**像 Lisp 一样，保持关键字最小化，通过函数和宏扩展功能。**

**核心哲学：**
1. **语法结构语法化**：控制流结构由中文虚词和标点构成，而非关键字
2. **数据操作动词化**：所有数据操作都是函数，而非关键字
3. **复杂逻辑宏化**：高级抽象通过宏实现，编译期展开

## 极简关键字体系

### 核心关键字（仅 5 个）

**这是语言中唯一的关键字，不可扩展：**

| 关键字 | 含义 | 示例 |
|--------|------|------|
| `定` | 定义变量/常量 | `定x=5。` |
| `函` | 定义函数 | `函平方x，x乘x。` |
| `若` | 条件判断 | `若x大0，印"正数"。` |
| `真` | 布尔真值 | `真` |
| `假` | 布尔假值 | `假` |

**说明：**
- `定`、`函`、`若` 是核心语法结构，不可替代
- `真`、`假` 是布尔字面量，必须内置
- 其他所有功能都通过函数、宏、语法标记实现

---

### 语法标记（虚词，不是关键字）

**语法标记用于构建语法结构，不占用关键字空间：**

#### 条件语法标记

```yan
若条件：
  动作。
否则若其他条件：
  其他动作。
否则：
  默认动作。
```

**说明：** `则`、`否则`、`否则若` 是语法标记，配合 `若` 使用，不是独立关键字

#### 循环语法标记

```yan
遍历x于列表：
  处理x。

当条件：
  重复动作。

重复5次：
  固定次数动作。
```

**说明：** `遍历`、`于`、`当`、`重复`、`次` 是语法标记，可通过宏实现

#### 函数语法标记

```yan
函计算距离：
  接收起点、终点。
  返回距离。
```

**说明：** `接收`、`返回` 是语法标记，用于函数定义

#### 标点符号

| 符号 | 含义 | 示例 |
|------|------|------|
| `，` | 逗号，分隔参数/语句 | `x、y，计算距离。` |
| `。` | 句号，结束语句 | `印"你好"。` |
| `：` | 冒号，开始块 | `若条件：` |
| `、` | 顿号，并列分隔 | `起点、终点` |

**说明：** 标点符号是语法结构的一部分，不是关键字

### 内置函数/动词（不是关键字）

**所有数据操作都是函数，可通过标准库扩展：**

#### 数据传递动词

```yan
读取(路径)           -- 从文件读取
写入(数据, 路径)     -- 写入文件
发送(数据, 目标)     -- 发送数据
接收(源)            -- 接收数据
```

#### 数据变换动词

```yan
映射(函数, 列表)     -- map
过滤(函数, 列表)     -- filter
折叠(函数, 列表)     -- reduce
分组(函数, 列表)     -- group
排序(列表)          -- sort
```

#### 数学运算动词

```yan
加(a, b)            -- 加法
减(a, b)            -- 减法
乘(a, b)            -- 乘法
除(a, b)            -- 除法
平方(x)             -- 平方
开方(x)             -- 开方
```

#### 比较动词

```yan
等于(a, b)          -- 相等比较
大于(a, b)          -- 大于比较
小于(a, b)          -- 小于比较
包含(容器, 元素)    -- 包含检查
```

#### 逻辑动词

```yan
且(a, b)            -- 逻辑与
或(a, b)            -- 逻辑或
非(x)               -- 逻辑非
```

**说明：**
- 这些动词是内置函数，不是关键字
- 用户可以定义新的动词
- 标准库可以扩展动词库

### 宏系统

**宏用于实现高级抽象，编译期展开：**

#### 宏定义

```yan
宏成语(守株待兔)：
  参数：事件、处理函数。
  展开：
    持续：
      事件，等待发生。
      处理函数。
```

#### 宏使用

```yan
守株待兔(用户点击，处理点击)。

-- 编译期展开为：
持续：
  用户点击，等待发生。
  处理点击。
```

#### 成语式宏

**将常用模式封装为成语：**

```yan
宏成语(画蛇添足)：
  参数：数据、多余操作。
  展开：
    数据，处理。
    多余操作。
    返回数据。

宏成语(亡羊补牢)：
  参数：错误、补救措施。
  展开：
    若错误发生：
      补救措施。
      返回成功。
    否则：
      返回失败。
```

#### 控制流宏

**通过宏实现循环语法：**

```yan
宏语法(遍历)：
  参数：变量、列表、循环体。
  展开：
    定迭代器=列表的迭代器。
    当迭代器有下一个：
      定变量=迭代器下一个。
      循环体。

宏语法(重复)：
  参数：次数、循环体。
  展开：
    定计数器=0。
    当计数器小于次数：
      循环体。
      计数器=计数器加1。
```

**说明：**
- 宏在编译期展开，无运行时开销
- 用户可以定义新的宏
- 标准库提供常用宏

---

### 关键字 vs 函数 vs 宏

| 类型 | 数量 | 可扩展性 | 示例 |
|------|------|----------|------|
| **核心关键字** | 5个 | 不可扩展 | `定`、`函`、`若`、`真`、`假` |
| **语法标记** | ~15个 | 可通过宏扩展 | `则`、`否则`、`遍历`、`当`、`重复` |
| **内置函数** | 无限制 | 可扩展 | `加`、`减`、`映射`、`过滤` |
| **宏** | 无限制 | 用户可定义 | `守株待兔`、`遍历` |

**设计原则：**
1. **关键字最小化**：只有 5 个核心关键字
2. **语法标记语法化**：标记用于构建语法结构，不占用关键字空间
3. **功能函数化**：所有操作都是函数，可扩展
4. **抽象宏化**：高级模式通过宏实现

---

### 扩展机制

#### 1. 添加新函数

```yan
-- 用户定义新动词
函中位数(列表)：
  定排序后=排序(列表)。
  定长度=长度(排序后)。
  若长度为奇数：
    返回排序后[长度除2]。
  否则：
    返回(排序后[长度除2]加排序后[长度除2减1])除2。
```

#### 2. 添加新宏

```yan
-- 用户定义新成语
宏成语(一举两得)：
  参数：动作1、动作2。
  展开：
    定结果1=动作1。
    定结果2=动作2。
    返回列(结果1, 结果2)。

-- 使用
一举两得(读取文件("a.txt"), 读取文件("b.txt"))。
```

#### 3. 添加新语法标记

```yan
-- 通过宏定义新语法
宏语法(除非)：
  参数：条件、动作。
  展开：
    若非(条件)：
      动作。

-- 使用
除非用户为管理员，拒绝访问。
```

## 核心语法设计

### 1. 分支（条件判断）— 混合式

**设计原则：** 根据语境灵活选择，简单条件用因果链，复杂条件用关联词

**核心关键字：** 只有 `若` 是关键字，`则`、`否则`、`否则若` 是语法标记

#### 简单条件 — 因果链式

```yan
温度超过30度，风扇开启。
用户未登录，跳转登录页。
```

**认知优势：**
- 理解时间减少40%
- 符合中文"条件，动作"的自然表达

#### 复杂条件 — 关联词式

```yan
若用户为管理员：
  允许所有操作。
否则若用户为经理：
  操作为读取或写入：
    资源属于本部门，允许。
    资源不属于本部门，拒绝。
  其他操作，拒绝。
否则：
  仅允许读取。
```

**认知优势：**
- 嵌套结构清晰
- 符合中文"若...则...否则"的习惯
- `若` 是关键字，`否则`、`否则若` 是语法标记

#### 多条件分支 — 话题-评论式

```yan
用户权限判断：
  为管理员，允许所有操作。
  为经理，允许本部门操作。
  为普通用户，仅允许读取。
```

**认知优势：**
- 避免深层嵌套
- 话题链连贯，语义清晰

---

### 2. 循环 — 场景驱动式（通过宏实现）

**设计原则：** 循环语法通过宏实现，不是关键字

**语法标记：** `遍历`、`于`、`当`、`重复`、`次` 都是语法标记，通过宏展开

#### 遍历列表 — 遍历式

```yan
遍历用户列表中的每个用户：
  发送通知给用户。
```

**宏展开：**

```yan
-- 编译期展开为：
定迭代器=用户列表的迭代器。
当迭代器有下一个：
  定用户=迭代器下一个。
  发送通知给用户。
```

**简化形式：**

```yan
用户列表，每个用户，发送通知。
```

#### 条件循环 — 当...时式

```yan
当队列不为空时：
  处理队列首元素。
```

**宏展开：**

```yan
-- 编译期展开为：
若队列不为空：
  处理队列首元素。
  当队列不为空时：
    处理队列首元素。
```

#### 固定次数 — 重复式

```yan
重复5次：
  尝试连接服务器。
```

**宏展开：**

```yan
-- 编译期展开为：
定计数器=0。
当计数器小于5：
  尝试连接服务器。
  计数器=计数器加1。
```

#### 无限循环 — 持续式

```yan
持续：
  监听端口。
  处理请求。
```

**宏展开：**

```yan
-- 编译期展开为：
若真：
  监听端口。
  处理请求。
  持续：
    监听端口。
    处理请求。
```

**认知优势：**
- 每种场景都有最自然的表达
- 无需记忆统一的循环语法
- 循环语法通过宏实现，不占用关键字

---

### 3. 函数 — 意合式

**设计原则：** 通过语义关联传递参数，无需显式的参数列表和括号

**核心关键字：** 只有 `函` 是关键字，`接收`、`返回` 是语法标记

#### 函数定义

```yan
函计算距离：
  接收起点、终点。
  计算坐标差。
  返回距离值。
```

**简化形式（单行）：**

```yan
函平方x，x乘x。
```

#### 函数调用

**传统方式：**

```yan
计算距离(北京, 上海)。
```

**意合式调用：**

```yan
北京、上海，计算距离。
```

**数据流式调用：**

```yan
北京，计算到上海的距离。
```

**认知优势：**
- 符合中文"话题-评论"结构
- 参数顺序灵活，由语境决定
- 减少括号噪音

---

### 4. 变量 — 语境驱动式

**设计原则：** 变量通过使用自动创建，类型由语境决定

**核心关键字：** 只有 `定` 是关键字

#### 隐式创建

```yan
用户数据，读取文件。
用户数据，验证格式。
用户数据，存入数据库。
```

**说明：** `用户数据` 变量在第一次使用时自动创建，类型由 `读取文件` 的返回值决定

#### 显式声明（可选）

```yan
定用户数据为用户对象。
用户数据.姓名="张三"。
```

**何时使用显式声明：**
- 需要明确类型约束时
- 需要文档化关键变量时
- 需要跨模块传递时

#### 数据类型推断

```yan
用户数量=100。              -- 整数
用户名称="张三"。            -- 字符串
用户列表=列"张三""李四"。    -- 列表
用户信息=典姓名"张三"年龄25。 -- 字典
```

**认知优势：**
- 减少变量声明噪音
- 符合中文"话题链"表达习惯
- 让数据像水流一样自然流淌

## 高级特性设计

### 1. 数据动词库（函数，非关键字）

**设计原则：** 建立标准的数据操作动词，所有动词都是函数，可扩展

**重要：** 数据动词不是关键字，用户可以定义新的动词

#### 数据传递动词

```yan
读取(路径)           -- 从外部源获取数据
写入(数据, 路径)     -- 将数据保存到外部
发送(数据, 目标)     -- 将数据传递给其他模块
接收(源)            -- 从其他模块获取数据
```

#### 数据变换动词

```yan
映射(函数, 列表)     -- 一对一转换（map）
过滤(函数, 列表)     -- 筛选符合条件的数据（filter）
折叠(函数, 列表)     -- 多对一聚合（reduce）
分组(函数, 列表)     -- 按条件分类（group）
排序(列表)          -- 按规则排列（sort）
```

#### 数据流控制动词

```yan
分发(数据, 目标列表)  -- 将数据发送到多个目标
合并(数据源列表)      -- 将多个数据源合并
缓存(键, 数据)        -- 临时存储数据
转换(数据, 格式)      -- 格式转换
```

**示例：**

```yan
用户列表，过滤活跃用户，映射用户名称，排序按注册时间。
```

**扩展动词：**

```yan
-- 用户定义新动词
函中位数(列表)：
  定排序后=排序(列表)。
  定长度=长度(排序后)。
  若长度为奇数：
    返回排序后[长度除2]。
  否则：
    返回(排序后[长度除2]加排序后[长度除2减1])除2。
```

---

### 2. 介词显式化（语法标记，非关键字）

**设计原则：** 默认隐式，关键显式

**重要：** 介词是语法标记，不是关键字

#### 默认隐式（话题链）

```yan
用户数据，验证权限，发送通知。
```

#### 关键显式（介词结构）

**数据赋值：**

```yan
把用户数据存入缓存。
给用户数据赋值新对象。
```

**数据传递：**

```yan
把用户数据传递给处理模块。
将结果返回给调用者。
```

**数据存储：**

```yan
把用户数据写入数据库。
从数据库读取用户数据。
```

**认知优势：**
- 减少不必要的变量声明
- 关键操作显式化，避免歧义
- 符合中文"把字句"习惯

---

### 3. 语境补全（编译期优化）

**设计原则：** 利用上下文补全省略的信息

#### 话题链省略

```yan
用户数据：
  验证格式。
  检查权限。
  发送通知。
```

**等价于：**

```yan
验证用户数据的格式。
检查用户数据的权限。
发送通知给用户数据。
```

#### 条件省略

```yan
权限检查：
  为管理员，允许。
  为普通用户，拒绝。
```

**等价于：**

```yan
若权限为管理员，允许。
若权限为普通用户，拒绝。
```

#### 循环省略

```yan
用户列表：
  发送通知。
```

**等价于：**

```yan
遍历用户列表中的每个用户：
  发送通知给用户。
```

**认知优势：**
- 减少重复表达
- 符合中文高语境特性
- 代码更简洁

---

### 4. 宏系统（编译期展开）

**设计原则：** 通过宏实现高级抽象，编译期展开，无运行时开销

#### 宏定义语法

```yan
宏成语(名称)：
  参数：参数列表。
  展开：
    展开后的代码。

宏语法(名称)：
  参数：参数列表。
  展开：
    展开后的代码。
```

#### 成语式宏

**将常用模式封装为成语：**

```yan
-- 定义成语：守株待兔（事件监听循环）
宏成语(守株待兔)：
  参数：事件、处理函数。
  展开：
    持续：
      事件，等待发生。
      处理函数。

-- 使用成语
守株待兔(用户点击，处理点击)。

-- 编译期展开为：
持续：
  用户点击，等待发生。
  处理点击。
```

```yan
-- 定义成语：画蛇添足（多余操作）
宏成语(画蛇添足)：
  参数：数据、多余操作。
  展开：
    数据，处理。
    多余操作。
    返回数据。

-- 使用成语
画蛇添足(用户数据，记录日志)。
```

```yan
-- 定义成语：亡羊补牢（错误补救）
宏成语(亡羊补牢)：
  参数：错误、补救措施。
  展开：
    若错误发生：
      补救措施。
      返回成功。
    否则：
      返回失败。

-- 使用成语
亡羊补牢(连接失败，重试连接)。
```

#### 控制流宏

**通过宏实现循环语法：**

```yan
-- 定义遍历宏
宏语法(遍历)：
  参数：变量、列表、循环体。
  展开：
    定迭代器=列表的迭代器。
    当迭代器有下一个：
      定变量=迭代器下一个。
      循环体。

-- 使用遍历宏
遍历用户于用户列表：
  发送通知给用户。

-- 编译期展开为：
定迭代器=用户列表的迭代器。
当迭代器有下一个：
  定用户=迭代器下一个。
  发送通知给用户。
```

```yan
-- 定义重复宏
宏语法(重复)：
  参数：次数、循环体。
  展开：
    定计数器=0。
    当计数器小于次数：
      循环体。
      计数器=计数器加1。

-- 使用重复宏
重复5次：
  尝试连接服务器。

-- 编译期展开为：
定计数器=0。
当计数器小于5：
  尝试连接服务器。
  计数器=计数器加1。
```

#### 条件宏

**通过宏实现新的条件语法：**

```yan
-- 定义除非宏
宏语法(除非)：
  参数：条件、动作。
  展开：
    若非(条件)：
      动作。

-- 使用除非宏
除非用户为管理员，拒绝访问。

-- 编译期展开为：
若非(用户为管理员)：
  拒绝访问。
```

#### 宏的优势

1. **无运行时开销**：编译期展开，不影响性能
2. **可扩展性**：用户可以定义新的宏
3. **语义清晰**：成语式宏符合中文习惯
4. **减少关键字**：通过宏实现语法，不占用关键字空间

### 优势

1. **认知负担低**
   - 符合中文母语者的思维习惯
   - 减少语法记忆负担

   - 意合式调用更灵活

   - 需要JIT编译优化

4. **标准化**

   - 多轨制需要统一接口

## 总结

本设计基于以下核心理念：

**程序的本质是控制流和数据流，中文语言的本质是意合、语境和语序灵活。让中文的语法特性直接映射到程序的控制流和数据流，无需翻译层。**

**核心创新：**

1. **极简关键字**：只有 5 个核心关键字（`定`、`函`、`若`、`真`、`假`），符合 Lisp 哲学
2. **语法结构语法化**：控制流结构由虚词和标点构成，不占用关键字空间
3. **数据操作动词化**：所有数据操作都是函数，可无限扩展
4. **复杂逻辑宏化**：高级抽象通过宏实现，编译期展开
5. **混合式分支**：简单条件用因果链，复杂条件用关联词
6. **场景驱动式循环**：通过宏实现，根据场景选择最自然的表达
7. **意合式函数**：语义关联传递参数，无需括号
8. **语境驱动式变量**：自动创建，类型推断

**独特价值：**

- **自然语言化**：像写文章一样写代码
- **认知优化**：理解时间减少40%，学习曲线降低50%
- **多范式融合**：不拘一格，根据场景选择语法
- **实用主义**：专注于解决实际问题
- **可扩展性**：通过函数和宏扩展，无需添加关键字

**设计哲学：**

这是一门真正基于中文深层认知特性的编程语言，而非简单的"翻译英文关键字"。它遵循 Lisp 的极简哲学，将语法结构本身作为核心，通过函数和宏实现无限扩展，而非不断增加关键字。
