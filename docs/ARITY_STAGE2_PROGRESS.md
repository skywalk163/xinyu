**新增方法**：

**核心逻辑**：
```python
def _collect_args_by_arity(self, arity: Arity) -> List[ASTNode]:
    args = []
    while not self._is_at_end():
        # 检查是否应该停止收集
        if self._should_stop_collecting_args():
            break
        # 检查元数是否已满足
        if arity.should_stop_collecting(len(args)):
            break
        # 解析参数
        arg = self._parse_primary()
        args.append(arg)
    return args
```

## ⚠️ 遇到的问题

### 问题：文件编码问题

**现象**：
- parser.py文件包含中文全角标点符号（（）、：、，、。）
- Python解释器无法识别这些字符
- 导致SyntaxError

**影响**：
- 无法导入parser模块
- 测试无法运行
- 阻塞后续开发

**尝试的解决方案**：
1. ❌ 批量替换全角括号 → 破坏了三引号字符串
2. ❌ 批量替换所有中文标点 → 破坏了文件结构
3. ✅ 从git恢复文件

**根本原因**：
- parser.py文件在之前的编辑中引入了中文全角标点
- 这些标点在docstring中，但Python解释器无法识别

---

### 方案1：手动修复（推荐）

**步骤**：
