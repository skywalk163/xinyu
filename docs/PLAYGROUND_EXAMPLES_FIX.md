## 🔧 修复的问题

### 1. variables示例 - 布尔值问题

**问题**：`真值`和`假值`被翻译成`真`和`假`，但Python中应该是`True`和`False`

**修复**：
```yan
# 修改前
定义 布尔值 = 真值。

# 修改后
定义 布尔值 = True。
```

### 2. function示例 - 多参数函数调用问题

**问题**：元数驱动解析无法正确处理未注册函数的多参数调用

**修复**：简化示例，避免多参数函数调用
```yan
# 修改前
定义 两数之和 = 函 a b：
  返回 a 相加 b。
。
打印 两数之和 10 20。

# 修改后
定义 a = 10。
定义 b = 20。
定义 和 = a 相加 b。
打印 "10 + 20 = " 和。
```

### 3. condition示例 - ELIF语法不支持

**问题**：解析器不支持`可选`（ELIF）语法

**修复**：使用嵌套的if-else
```yan
# 修改前
如果 成绩 大于等于 90 那么：
  打印 "优秀"。
可选 成绩 大于等于 80 那么：
  打印 "良好"。
否则：
  打印 "不及格"。
。

# 修改后
如果 成绩 大于等于 90 那么：
  打印 "优秀"。
否则：
  如果 成绩 大于等于 80 那么：
    打印 "良好"。
  否则：
    打印 "不及格"。
  。
。
```

### 4. loop和list示例 - list函数参数问题

**问题**：Python的`list()`函数只接受1个参数

**修复**：使用列表字面量语法
```yan
# 修改前
定义 水果 = 列表 "苹果" "香蕉" "橘子"。

# 修改后
定义 水果 = ["苹果", "香蕉", "橘子"]。
```

### 5. fibonacci示例 - 递归调用问题

**问题**：递归函数调用时参数传递错误

**修复**：使用迭代版本
```yan
# 修改前（递归）
定义 斐波那契 = 函 n：
  如果 n 小于 2 那么：
    返回 n。
  。
  返回 斐波那契 n 相减 1 相加 斐波那契 n 相减 2。
。

# 修改后（迭代）
定义 a = 0。
定义 b = 1。
遍历 i 于 范围 1 11：
  打印 a。
  定义 temp = a。
  定义 a = b。
  定义 b = temp 相加 b。
。
```

### 6. math示例 - 多参数打印和负数问题

**问题**：
- `打印 "a = " a ", b = " b` 被错误解析
- `绝对值 -5` 被解析为减法

**修复**：简化输出，使用变量
```yan
# 修改前
打印 "a = " a ", b = " b。
打印 "绝对值(-5) = " 绝对值 -5。

# 修改后
打印 "a 相加 b = " a 相加 b。
定义 x = -5。
打印 "绝对值(-5) = " 绝对值 x。
```

## 🎯 经验总结

### 1. 心语语法限制

**当前支持**：
- ✅ 单参数函数调用
- ✅ 嵌套的if-else
- ✅ 列表字面量 `[1, 2, 3]`
- ✅ 字典字面量 `{"key": value}`
- ✅ 基本运算符

**当前不支持**：
- ❌ 多参数函数调用（需要元数注册）
- ❌ ELIF语法（可选）
- ❌ 复杂的多参数打印
- ❌ 负数字面量作为函数参数

### 2. 推荐的写法

**函数调用**：
```yan
# 单参数函数
平方 5

# 多参数 - 分步处理
定义 a = 10。
定义 b = 20。
定义 和 = a 相加 b。
```

**条件判断**：
```yan
# 使用嵌套if-else
如果 条件1 那么：
  分支1。
否则：
  如果 条件2 那么：
    分支2。
  否则：
    分支3。
  。
。
```

**列表创建**：
```yan

定义 列表 = [1, 2, 3, 4, 5]。
```
