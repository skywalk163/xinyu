# REPL历史记录增强功能文档

## 概述

本增强功能为心语编程语言的REPL（交互式解释器）添加了强大的历史记录管理功能。基于tasks.md文档中的REQ-004需求，实现了历史记录的搜索、编辑、持久化存储和导入导出功能。

## 主要功能

### 1. 增强的历史记录管理器 (`HistoryManager`)

#### 核心特性
- **智能命令类型检测**: 自动识别命令类型（表达式、语句、定义、导入、控制、调试、帮助等）
- **丰富的元数据**: 记录执行时间、成功状态、标签、自定义元数据
- **多种存储方式**: 支持内存存储和SQLite数据库存储
- **强大的搜索过滤**: 支持关键词搜索、命令类型过滤、时间范围过滤、标签过滤
- **编辑和重新执行**: 支持编辑历史命令并重新执行
- **导入导出**: 支持JSON和CSV格式的导入导出
- **统计信息**: 提供详细的统计报告

#### 数据模型
```python
@dataclass
class HistoryEntry:
    timestamp: datetime          # 时间戳
    command: str                # 命令内容
    result: Optional[str]       # 执行结果
    command_type: CommandType   # 命令类型
    execution_time: Optional[float]  # 执行时间（秒）
    success: bool              # 是否成功
    tags: List[str]            # 标签列表
    metadata: Dict[str, Any]   # 元数据
```

### 2. REPL集成

#### 新增命令
```
历史 列表 [数量]          - 显示历史记录（默认显示最近10条）
历史 搜索 <关键词>        - 搜索历史记录
历史 过滤 <类型>          - 按类型过滤历史记录
历史 统计                 - 显示历史记录统计信息
历史 导出 <文件>          - 导出历史记录到文件
历史 导入 <文件>          - 从文件导入历史记录
历史 编辑 <编号> <新命令> - 编辑历史记录并重新执行
历史 清除                 - 清除所有历史记录
```

#### 命令类型说明
- `expression`  - 表达式求值
- `statement`   - 语句执行
- `definition`  - 定义（函数、变量等）
- `import`      - 导入语句
- `control`     - 控制语句（if、for等）
- `debug`       - 调试命令
- `help`        - 帮助命令
- `other`       - 其他命令

### 3. 自动标签系统

系统会自动为命令添加标签，便于分类和过滤：
- `定义` - 包含"定义"的命令
- `条件` - 包含"如果"或"否则"的命令
- `循环` - 包含"循环"、"当"或"为"的命令
- `导入` - 包含"导入"的命令
- `输出` - 包含"打印"的命令
- `输入` - 包含"输入"的命令
- `调试` - 包含"调试"的命令

## 技术实现

### 1. 架构设计

```
EnhancedREPL
    ├── HistoryManager (增强历史记录管理)
    │   ├── 内存存储 (pickle序列化)
    │   ├── 数据库存储 (SQLite，可选)
    │   ├── 搜索过滤引擎
    │   └── 统计报告生成器
    ├── CodeCompletionProvider (代码补全)
    └── SyntaxHighlighter (语法高亮)
```

### 2. 持久化存储

- **默认存储**: 使用pickle序列化到 `~/.xinyu_history` 文件
- **数据库存储**: 可选SQLite数据库存储，支持更复杂的查询
- **导入导出**: 支持JSON和CSV格式，便于数据迁移和分析

### 3. 性能优化

- **懒加载**: 历史记录按需加载，减少内存占用
- **索引优化**: 数据库模式下使用索引加速搜索
- **分页查询**: 支持限制返回结果数量
- **缓存机制**: 常用查询结果缓存

## 配置选项

### REPL初始化参数

```python
repl = EnhancedREPL(
    compiler,                    # 编译器实例
    history_file=".xinyu_history",  # 历史记录文件路径
    use_database=False,         # 是否使用数据库存储
    max_size=1000               # 最大历史记录数量
)
```

### 历史记录管理器配置

```python
from src.repl.history_manager import HistoryManager

manager = HistoryManager(
    max_size=1000,              # 最大记录数
    history_file=Path.home() / '.xinyu_history',  # 存储文件
    use_database=False          # 使用数据库存储
)
```

## 故障排除

### 常见问题

1. **历史记录文件损坏**
   ```
   解决方案: 删除损坏文件，系统会自动创建新文件
   rm ~/.xinyu_history
   ```

2. **数据库连接失败**
   ```
   解决方案: 切换到文件存储模式
   repl = EnhancedREPL(compiler, use_database=False)
   ```

3. **导入导出格式错误**
   ```
   解决方案: 检查文件格式，确保是有效的JSON或CSV
   ```
