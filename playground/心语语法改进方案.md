# 心语语法改进方案

## 当前问题分析

### 1. 函数表达式参数不支持
**当前语法**：`函数名 参数1 参数2`
**问题**：不支持 `函数名 (表达式) 参数` 形式
**示例**：`汉诺塔 (减 n 1) 源 辅助 目标` 无法解析

### 2. 递归调用限制
**当前语法**：需要中间变量
**问题**：`阶乘 (减 n 1)` 无法直接调用
**变通方案**：
```javascript
定义 减一 = 减 n 1。
阶乘 减一。
```

### 3. 字符串连接复杂
**当前语法**：`连 "移动盘子从 " 源 " 到 " 目标`
**问题**：参数顺序和空格处理复杂

### 4. 变量作用域问题
**当前问题**：循环内定义同名变量可能导致冲突

### 5. 语句块结束标记不清晰
**当前语法**：使用句号`.`结束，嵌套时容易混淆

## 改进方案

### 1. 支持函数表达式参数
**修改解析器**：在 `_parse_term()` 方法中支持括号表达式作为参数

**实现思路**：
1. 在解析函数调用时，检查下一个token是否是左括号`(`
2. 如果是，解析括号内的表达式作为第一个参数
3. 继续解析其他参数

**语法示例**：
```javascript
# 支持括号表达式作为参数
汉诺塔 (减 n 1) 源 辅助 目标

# 支持多个括号表达式
计算 (加 a b) (乘 c d)
```

### 2. 改进递归支持
**修改函数调用解析**：允许函数调用自身

**实现思路**：
1. 在函数定义时记录函数名
2. 在函数体解析时，允许调用同名函数
3. 处理递归调用的参数传递

**语法示例**：
```javascript
定义 阶乘 = 函 n：
  如果 等 n 0 那么：
    返回 1。
  。
  否则：
    返回 n 相乘 阶乘 (减 n 1)。
  。
。
```

### 3. 简化字符串连接
**新增语法糖**：支持更自然的字符串连接

**实现思路**：
1. 支持 `"字符串" + 变量` 语法
2. 或者改进 `连` 函数的参数处理

**语法示例**：
```javascript
# 方案1：使用+操作符
打印 "移动盘子从 " + 源 + " 到 " + 目标。

# 方案2：改进连函数
打印 连("移动盘子从 ", 源, " 到 ", 目标)。
```

### 4. 增强变量作用域
**实现局部变量**：在函数和循环内创建局部作用域

**实现思路**：
1. 函数参数和函数内定义的变量是局部的
2. 循环内定义的变量在循环外不可访问
3. 支持变量遮蔽（shadowing）

**语法示例**：
```javascript
定义 全局 = 10。

定义 测试 = 函 x：
  定义 局部 = 20。  # 局部变量
  返回 全局 相加 局部 相加 x。
。

遍历 i 于 范围 1 5：
  定义 临时 = i 相乘 2。  # 循环局部变量
  打印 临时。
。
# 这里不能访问 临时
```

### 5. 改进语句块语法
**使用更清晰的结束标记**：`结束` 关键字

**实现思路**：
1. 使用 `结束` 代替句号 `.` 结束语句块
2. 支持嵌套语句块的清晰标记

**语法示例**：
```javascript
如果 条件 那么：
  # 语句块
  打印 "条件成立"。
结束

定义 函数 = 函 参数：
  # 函数体
  返回 参数 相乘 2。
结束

遍历 元素 于 列表：
  # 循环体
  打印 元素。
结束
```

## 具体实现步骤

### 步骤1：修改解析器支持括号表达式参数
1. 在 `_parse_function_call()` 方法中添加括号表达式支持
2. 修改 `_collect_args_by_arity()` 方法处理括号表达式
3. 更新AST节点支持表达式参数

### 步骤2：改进递归支持
1. 在语义分析阶段处理函数名解析
2. 允许函数体内引用自身
3. 处理递归调用的类型检查

### 步骤3：简化字符串操作
1. 添加字符串连接操作符`+`的支持
2. 或者改进 `连` 函数的实现

### 步骤4：增强变量作用域
1. 在语义分析阶段创建作用域链
2. 实现变量查找规则
3. 处理变量遮蔽

### 步骤5：改进语句块语法
1. 修改词法分析器识别`结束`关键字
2. 更新解析器使用`结束`作为块结束标记
3. 保持向后兼容性（仍然支持句号`.`）

## 向后兼容性考虑

### 保持兼容的语法
1. 仍然支持现有的句号`.`结束语法
2. 现有的函数调用语法不变
3. 现有的变量定义语法不变

### 渐进式改进
1. 先实现括号表达式参数支持
2. 然后改进递归支持
3. 最后添加新的语法糖

## 测试用例

```javascript
定义 加 = 函 a b：
  返回 a 相加 b。
。

打印 "加(2+3, 4+5) = " 加 (2 相加 3) (4 相加 5)。
```

### 测试2：递归函数
```javascript
定义 斐波那契 = 函 n：
  如果 等 n 0 那么：
    返回 0。
  。
  如果 等 n 1 那么：
    返回 1。
  。
  返回 加 (斐波那契 (减 n 1)) (斐波那契 (减 n 2))。
。

打印 "斐波那契(10) = " 斐波那契 10。
```

### 测试3：汉诺塔（纯言风格）
```javascript
定义 汉诺塔 = 函 n 源 目标 辅助：
  如果 等 n 1 那么：
    打印 "移动盘子从 " + 源 + " 到 " + 目标。
    返回。
  。
  汉诺塔 (减 n 1) 源 辅助 目标。
  打印 "移动盘子从 " + 源 + " 到 " + 目标。
  汉诺塔 (减 n 1) 辅助 目标 源。
。

打印 "汉诺塔 3 个盘子："。
汉诺塔 3 "A" "C" "B"。
```

## 预期效果

### 改进后的语法优势
1. **更自然**：支持数学表达式作为函数参数
2. **更简洁**：减少中间变量定义
3. **更强大**：支持真正的递归算法
4. **更清晰**：语句块结束标记明确
5. **更灵活**：变量作用域管理更好

### 对现有代码的影响
1. 现有代码仍然可以工作
2. 新语法是可选的增强功能
3. 可以逐步迁移到新语法
